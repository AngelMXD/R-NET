# 4.2 ❯ 📨 Guia para Adjuntar Tickets y Casos  

Los casos y tickets se adjuntan en un canal llamado: **"✰₊˚📦．adjuntar・pruebas"**. Cada ticket tiene su propocito, pero hay 3 temas que son "frecuentes" que son: Reportes, Sanciones y Consultas. Debido a que hay estos 3 tipos de tickets puedes utilizar 2 formatos y serian los siguientes.

## ・Para adjuntar tickets:  
**Adjuntar los tickets es muy importante para mantener el registro. Aquí abajo podrás ver cómo debería ser. Puedes decorarlo a tu estilo, pero debe seguir el siguiente formato. Este tipo de formato es como "ticket informativo"**  

+++ :icon-archive: Ejemplo
- Ticket: #1000
- ID del usuario: 684447295420629032
- Razón: El USR. abrio ticket para informacion de como aplicar a una postulacion de staff. Se le comenta que para aplicar, tendra que estar atento a las convocatorias que se hacen en el canal de anuncios.
+++ :icon-apps: Formato
```
- Ticket: [Número del ticket]  
- ID del usuario: [Aquí tendrás que poner el **ID** del usuario que creó el ticket]  
- Razón: [Aquí tendrás que redactar el **motivo** por el cual se abrió el ticket.]  
```

+++ :icon-book: Informacion
El formato del "Ticket informativo" tiene que incluir la siguiente informacion:
  - Numero del ticket
  - ID del usuario que abrio el ticket
  - Motivo por el cual se abrio el ticket
+++ 	:icon-megaphone: Importante
Siempre habra reportes de un usuario dentro de un ticket ya que es la forma correcte de llevar un seguimiento. En el caso que un usuario abra un ticket para **"Sancionar a otro usuario"** tendras que adjuntar el 'Case' que genero el bot **<span style="color:red;">UnbelievaBoat</span>**. ## ・Para adjuntar casos:  
**Este tipo de formato sera diferente, ya que no huvo un ticket previo y se sanciono de manera interna, asi que tendras que adjuntar el 'Case' correspondiente que genera el bot **<span style="color:red;">UnbelievaBoat</span>**. Aqui te enseñare un ejemplo de como seria el formato**  


+++ :icon-apps: Formato
```
- **Caso:** [Número del caso]  
- **Tipo de sanción:** [Aquí tendrás que escribir el tipo de sanción aplicada al usuario].  
- **Motivo:** [Tendrás que redactar el motivo por el cual se sancionó al usuario].  
```

+++ :icon-archive: Ejemplo

Ya teniendo el formato listo, el mensaje debe de quedar así. Cabe recalcar que este seria la base, asi que puedes decorarlo a tu estilo mientras mantengas la informacion relevante

![](/assets/ejemplo-ticket.webp){.rounded-lg width="600"}


!!!Danger **Importante**  
Cada sanción debe contar con una prueba verídica. Si no se tiene, pueden surgir problemas, por lo que tendrás que subir los archivos pertinentes, como por ejemplo alguna imagen o un video grabado.
